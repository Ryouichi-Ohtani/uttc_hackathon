# 要件

## 必須要件

### 技術スタック

**Web開発**

- **バックエンド**: Go
- **フロントエンド**: React
- **AI API**: Google Gemini API

**AI開発**

- **バックエンド**: Python
- **フロントエンド**: React
- **AI API**: Google Gemini API

### 必須機能

**Webコース**

- **ユーザー登録・認証**: 基本的なユーザー管理機能
- **商品出品**: 商品情報入力
- **商品購入**: 購入フロー（決済API連携は不要）
- **DM機能**: ユーザー間でのメッセージ交換
- **Gemini API連携**: 商品説明の自動生成や質問応答機能
- **デプロイ:** バックエンドはCloudRun, フロントエンドはVercelを用いる。また、DBに関してはCloudSQLを用いる

**AIコース**

- **ユーザー登録・認証**: 基本的なユーザー管理機能
- **商品出品**: 商品情報入力
- **商品購入**: 購入フロー（決済API連携は不要）
- **レコメンド:** 機械学習を用いて商品のレコメンドを行う
- **デプロイ:** バックエンドはCloudRun, フロントエンドはVercelを用いる、DBに関してはCloudSQLを用いる

## 発展的な実装の例

### 中級レベル

- **いいね機能の実装**
    - 商品をいいねする機能
- **インフラ層での実装**
    - CDNを使用した画像配信最適化
    - 適切な権限管理によるセキュリティ強化
- **認証・認可の強化**
    - JWT認証の実装
    - ロールベースアクセス制御
- **データベース最適化**
    - 適切なインデックス設計
    - クエリパフォーマンスの向上
- **多様な通信方式**
    - WebSocket、gRPC、RESTの使い分け
- **カスタムレコメンドモデル**
    - Kaggleデータセットを活用した独自モデル開発
- **画像・テキスト解析**
    - 商品画像とキャプションの関係性分析
    - 不適切コンテンツの自動検出
- **高度なAI連携**
    - LangChain/LangGraphを使用した複雑なワークフロー実装
- **テスト整備**
    - 単体テストを実装
    - CIまで整えられると良い

### 上級レベル

- **3Dモデル表示**: Three.jsやWebGLを使用した商品の立体表示
- **高度な分析機能**: ユーザー行動分析、売上予測
- **リアルタイム機能**: ライブ配信での商品紹介、リアルタイム入札
- **多言語対応**: 国際展開を見据えたi18n実装
- **マイクロサービス**: 複数サービスの連携による大規模アーキテクチャ

### 超上級レベル

- **独創的な機能**: AR試着機能、音声検索、ブロックチェーン活用

## 評価項目

### **技術・実装**

- **アーキテクチャ:** 拡張性や保守性を考慮した、クリーンで合理的な設計がなされているか。（例: クリーンアーキテクチャ、責務の分離）
- **コード品質:** 可読性、一貫性があり、他の開発者が見ても理解しやすいコードが書けているか。
- **挑戦度:** 必須機能に加え、発展的な実装にどれだけ意欲的に挑戦したか。

### **完成度・UX**

- **必須機能の実装:** 要件定義された必須機能が、仕様通りに安定して動作するか。
- **UI/UXデザイン:** ユーザーにとって直感的で使いやすく、ストレスのない操作感を提供できているか。デザインの一貫性も評価します。
- **デモの完成度:** Demo Dayの発表において、主要な機能が一通り動作し、プロダクトの魅力を伝えられる状態にあるか。

### **テーマ性・独創性**

- **テーマの体現:** 「次世代フリマアプリ」というテーマをどう解釈し、プロダクトに落とし込んでいるか。
- **AIの活用価値:** Gemini APIや独自モデルを効果的に活用し、既存のフリマアプリにはない新しいユーザー体験や価値を創出できているか。（例: 単なる商品説明生成だけでなく、価格交渉の補助、出品禁止物の検知など）
- **アイデアの新規性:** 他のチームにはない、ユニークで創造的なアイデアが盛り込まれているか。

### **プレゼンテーション**

- **魅力の伝達:** 開発したプロダクトのコンセプト、技術的な挑戦、そして最もアピールしたい点を、限られた時間の中で分かりやすく伝えられているか。
- **デモンストレーション:** プロダクトの魅力を効果的に見せるデモができているか。
- **質疑応答:** 審査員からの質問に対し、的確かつ自信をもって答えられているか。

# 具体的な流れ

## ① 設計

### TODO

- **UI/UX設計:**
    - **WHAT:** 主要画面のワイヤーフレーム（画面遷移図）を作成し、ユーザー体験の流れを固める。
    - **TOOL:** Figma, Miro, draw.io など
    - **OUTPUT:** ワイヤーフレーム、画面遷移図
- **DB設計:**
    - **WHAT:** 必要なテーブル、カラム、リレーションシップを定義する。
    - **TOOL:** ER図作成ツール (e.g., dbdiagram.io), スプレッドシート
    - **OUTPUT:** ER図、テーブル定義書
- **API設計:**
    - **WHAT:** 必要なエンドポイント、リクエスト/レスポンスのデータ形式 (JSON) を定義する。
    - **TOOL:** OpenAPI (Swagger), Postman, スプレッドシート
    - **OUTPUT:** API仕様書

### レビュー

- **WHEN:** 上記の設計成果物が完成したタイミング
- **WHO:** メンター 及び チームメンバー（Weekly等でレビューをもらいましょう）
- **WHAT:** 設計に大きな問題がないか、実装に進む上での懸念点はないかを確認し、フィードバックを受けます。
